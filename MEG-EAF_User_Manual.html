System Overview

MEG-EAF is MyEden Group's employment application and appointment-letter system. It has two parts:

  • Applicant Form (index.html) — a public web page where job applicants fill in and submit their application. No login or account is required.
  • HR Dashboard (hr.html) — a login-protected page for HR/admin staff to review applications, complete internal Office Use details, and generate the official Appointment Letter.

Both pages are hosted together and share the same underlying data, stored securely online. Applicant submissions appear in the HR Dashboard automatically — no email or manual file transfer is needed.

Both files always carry the same version stamp (e.g. v.2026.07.22-01:00) so you can tell at a glance that they are aligned and from the same update.
Part 1 — Applicant Guide

A. Accessing the Application Form

Open the application link provided by MyEden Group in any web browser (desktop or mobile). The form loads immediately — there is no login step and no access code needed to open it.

B. Filling In the Form

The form is organised into clearly labelled sections. Fields marked with a red asterisk (*) are required.

  • Personal Particulars — name (English and Chinese, if applicable), NRIC or passport number, date of birth, gender, marital status, race, religion, number of children.
  • Address & Contact — home address, postcode/township, state, phone number, email address.
  • Family Information — add one row per family member using + Add family member.
  • Academic Achievements — add one row per qualification using + Add qualification.
  • Work Experience — add a block per previous employer.
  • Job Requirements — tick the position(s) you are applying for, grouped as:
    • A. Operations — Childcare Mentor, Kindergarten Teacher, Infant Childcare, Playgroup Mentor, or Others (please specify).
    • B. Office — Management, HR, Admin, Account, Account & Payroll, Content Creator, Marketing, or Others (please specify).
    Also indicate your desired working hours (Full Time, Part Time, Internship, Temporary, or Others).
  • Other Questions — language proficiency, special skills, and standard screening questions.
  • Declaration — read the declaration statement and tick to confirm before submitting.

C. Photos, ID Documents & Signature

You will be asked to provide:

  • A passport-style photo of yourself.
  • Front and back photos of your I/C (or your passport photo page, if you tick "Using Passport instead of I/C").
  • Your signature, drawn directly on screen using your finger, stylus, or mouse.

For each photo box, use the Camera button to take a photo directly, or Gallery to upload an existing image. Use Clear to remove and retake a photo. Position your I/C so it fills the frame guide as closely as possible before capturing, for the clearest result.

Photos are shown in full within their box, scaled to fit — if a photo appears with grey bands on the sides, this simply means the original picture wasn't perfectly cropped to the card; the full image is still there underneath.

D. Saving a Draft & Resuming Later

If you can't finish the form in one sitting:

  • Click 💾 Save Draft at any time. You'll see a brief confirmation ("Draft saved ✓") — there is nothing further to note down.
  • If you return to the same device and browser later using the same link, the form will automatically ask: "You have a saved draft — continue where you left off?" Tap Yes to restore everything, including your signature.
  • Even if you forget to click Save Draft, the form automatically keeps a local backup of your progress every couple of seconds, so switching tabs or closing by accident is usually recoverable on the same device.
Resuming on a different device. Because the form doesn't require an account, a saved draft can only be found automatically on the device you saved it from. If you need to continue on a different phone or computer, click 📂 Resume Draft and you will be asked for a draft code. If you don't have this code, simply contact MyEden Group HR — they can look up your draft by your name and read the code to you.

E. Local Backup (Download / Load JSON)

As an extra safety net independent of the online draft system, you can keep your own backup file on your device:

  • 💾 Download JSON — saves everything currently entered on the form as a small file (ending in .json) to your device's usual downloads folder. This file is for your own records; it is not sent anywhere.
  • 📂 Load from JSON — choose a previously downloaded file to restore all of that data back into the form. This will replace anything currently on screen, so use it at the start of a session.

After you submit your application, the confirmation screen also offers a "Download a copy (JSON)" button, so you can keep a personal record of exactly what was submitted.

F. Submitting the Application

Once every required field is filled in, click 📤 Submit Application. You will be asked to confirm, since the application cannot be edited once submitted. After confirming, a success screen appears — your application has been received, and you may safely close the page.

Part 2 — HR / Admin Staff Guide

A. Signing In

Open the HR Dashboard link (a separate address from the applicant form, usually ending in /hr.html). Sign in with the email and password given to you by your administrator. Your session stays active for the duration of your visit and refreshes automatically; use Sign Out when you're done, especially on a shared computer.

Staff accounts are created by the administrator directly — there is no self-registration.

B. Dashboard Overview

After signing in you'll see three tabs (Manage Staff is visible to administrators only):

  • 📥 Applications — every submitted application, newest first.
  • 📝 Drafts — applicants' in-progress, not-yet-submitted forms.
  • 👥 Manage Staff (admin only) — control what each staff member is allowed to do.

Use the search box at any time to filter the current tab by applicant name.

C. Applications Tab

Each application is shown as a card with the applicant's name, position applied for, submission date, and a status badge (PENDING or PROCESSED). Click a card to open the full application, exactly as the applicant filled it in, plus the internal Office Use section described next.

D. Office Use & Appointment Letter Details

Scroll to the green "For Office Use Only" section (visible only to staff with the appropriate permission). Complete the following before generating a letter:

  • Approved Job Title — choose from the Operations/Office list, or select Other to type a title manually.
  • Job Title Level — once a title is chosen, this list automatically shows that role's career levels (e.g. Kindergarten Associate → Kindergarten Mentor → Senior Kindergarten Mentor → Kindergarten Manager → Head of Kindergarten). The Resolved Position line beneath confirms exactly what will print on the letter.
  • Work Location — choose one of MyEden Group's branch addresses, or tick Other location to type one manually.
  • Letter Date — defaults to today; change if needed.
  • Reporting Officer's Name and Job Title, Salary in Wording (auto-generated from the salary figure, editable), and other appointment fields (start date, salary, working hours, etc.).
  • Appendix 1 (Job Description) — automatically fills in with the standard job description for the selected Job Title Level. Tick "Manually fill / edit Appendix 1" to unlock the box and write or adjust the description yourself; untick to restore the standard wording.
A live preview of the full Appointment Letter appears further down the page as you fill these fields in, so you can check the wording before printing.

E. Saving, Status & Deleting

  • 💾 Save Changes (top bar) or 💾 Save Draft (bottom action bar) — both save your Office Use edits back to the application record. Use whichever is more convenient while scrolling.
  • Mark Processed — flips the status badge; click again (now labelled Reopen) to revert to Pending.
  • 🗑 Delete — permanently removes the application. Only available to staff with delete permission, and always asks for confirmation first.

F. Drafts Tab

Shows every applicant's in-progress, unsubmitted form, along with their name, contact details, and a draft code. This code is what lets an applicant resume their draft on a different device: if someone calls or messages asking to continue on a new phone, look them up here by name and read them the code shown. Opening a draft here is read-only — it lets you check progress and help the applicant, but doesn't allow editing.

G. Printing & PDF Export

  • 🖶 Print / Save PDF — prints the complete application form as filled in, including the Office Use section and the full Job Description text. The Appointment Letter preview is intentionally excluded from this printout.
  • 📋 Print Appointment Letter — prints (or saves to PDF, via your browser's print dialog) only the official Appointment Letter, formatted with letterhead, the full contract clauses, salient-terms table, and Appendix 1 job description.
Use your browser's "Save as PDF" destination option in the print dialog if you want a PDF file instead of a physical printout.

H. Local Backup (Download / Load JSON)

Alongside the online record, you can keep or restore a local backup file of whichever application, draft, or file you currently have open:

  • 💾 Download JSON (in the top detail bar) — saves the currently open record as a .json file to your computer.
  • 📂 Load from JSON (at the top of the dashboard) — opens a file picker to load a previously downloaded backup file. The loaded record displays clearly labelled as "LOADED FROM FILE (not linked to database)" — you can view, print, and generate a letter from it, but Save/Status/Delete are disabled since it isn't connected to any live database record.

I. Manage Staff (Admin Only)

Administrators see a Manage Staff tab listing every staff account, each with tickable permission flags:

PermissionControls
View applicationsWhether they can see the Applications inbox at all.
Edit Office UseWhether they can fill in or change the Office Use section.
Generate lettersWhether the Appointment Letter section and print button are available to them.
Change statusWhether they can mark applications Processed/Pending.
Delete recordsWhether they can delete applications or drafts.

Changes take effect immediately — no need to log the staff member out. Administrators automatically hold every permission, and cannot accidentally remove their own admin rights without an explicit confirmation prompt.

J. Creating Staff Accounts

New staff accounts are created directly in the Supabase project console (not from within the HR Dashboard itself), since this requires administrator-level database access:

  1. Sign in to the Supabase project.
  2. Go to Authentication → Users → Add user → Create new user.
  3. Enter the staff member's email and a password, keeping Auto Confirm User ticked.
  4. The account appears automatically in the Manage Staff panel with safe default permissions, ready for you to adjust.

K. Troubleshooting (HR/Admin)

IssueWhat to do
"Email not confirmed" error at loginThe account was created without Auto Confirm ticked. In Supabase's SQL Editor, run: update auth.users set email_confirmed_at = now() where email_confirmed_at is null;
"Send confirmation email" fails in SupabaseExpected — sign-ups are intentionally disabled project-wide, which also blocks this email feature. Use the SQL fix above instead.
Forgot a staff passwordReset it directly in Supabase → Authentication → Users → select the user → Reset Password.
Appendix 1 text reverted after reopeningMake sure "Manually fill / edit Appendix 1" is ticked before typing custom text, and click Save Changes afterwards — the tick state is now saved together with the text.
Printed letter looks cut off or misalignedAlways judge print layout using your browser's own Print/Save-as-PDF dialog; other PDF viewers or converters may render the page differently.

Versioning & Change Log Convention

Both index.html and hr.html share a single version stamp, shown as MEG-EAF v.YYYY.MM.DD-HH:MM, updated together whenever either file changes — so matching stamps confirm the two files are aligned. This manual is issued against v.2026.07.22-01:00; if the deployed system shows a later stamp, a newer manual may be available.

系统概览

MEG-EAF 是 MyEden Group 的求职申请与聘书生成系统，分为两个部分：

  • 求职申请表格（index.html）— 公开的网页，供求职者填写并提交申请，无需注册或登入账户。
  • 人力资源后台（hr.html）— 需要登入的页面，供人力资源／管理员查看申请、填写内部资料，并生成正式的聘书。

两个页面共用同一套安全的线上数据库。求职者提交的申请会自动出现在人力资源后台中，无需透过电邮或人手传送文件。

两个文件的版本号会始终保持一致（例如 v.2026.07.22-01:00），方便您一眼确认两者是否为同一次更新。
第一部分 — 求职申请人指南

A. 如何进入申请表格

使用任何浏览器（电脑或手机）打开 MyEden Group 提供的申请链接，表格会立即载入 — 无需登入，也无需任何访问码。

B. 填写表格

表格分为多个清楚标示的部分。带有红色星号（*）的栏位为必填项目。

  • 个人资料 — 姓名（英文及中文，如适用）、身份证或护照号码、出生日期、性别、婚姻状况、种族、宗教、子女人数。
  • 地址与联络方式 — 家庭地址、邮政编码／市镇、州属、电话号码、电邮地址。
  • 家庭成员资料 — 点击 + Add family member 逐一添加家庭成员。
  • 学历资料 — 点击 + Add qualification 逐一添加学历。
  • 工作经验 — 为每一份previous工作添加一个区块。
  • 职位需求 — 勾选您所申请的职位，分为：
    • A. 营运类（Operations） — 幼儿导师（Childcare Mentor）、幼稚园老师（Kindergarten Teacher）、婴幼儿托育（Infant Childcare）、游戏班导师（Playgroup Mentor），或其他（请注明）。
    • B. 行政类（Office） — 管理层（Management）、人力资源（HR）、行政（Admin）、会计（Account）、会计与薪资（Account & Payroll）、内容创作（Content Creator）、市场行销（Marketing），或其他（请注明）。
    同时请注明期望的工作时间（全职、兼职、实习、临时，或其他）。
  • 其他问题 — 语言能力、特殊技能，以及一般筛选问题。
  • 声明 — 请阅读声明内容，并勾选确认后方可提交。

C. 照片、身份证件与签名

系统将要求您提供：

  • 一张个人证件照片。
  • 身份证正反两面的照片（若勾选 "Using Passport instead of I/C"，则改为护照资料页照片）。
  • 您的签名 — 可直接以手指、触控笔或鼠标在屏幕上签署。

在每个照片框内，点击 Camera 直接拍照，或点击 Gallery 从相册上传现有照片。点击 Clear 可清除并重新拍摄。拍摄时请尽量将身份证对齐框内的引导线，以获得最清晰的效果。

照片会以完整比例显示于框内，若照片两侧出现灰色留白，代表原始照片未完全贴合边框裁切，完整照片仍会保留在系统中。

D. 保存草稿与稍后继续

如果您无法一次填完整份表格：

  • 随时点击 💾 Save Draft 保存草稿，系统会短暂显示确认信息（"Draft saved ✓"），无需记下任何代码。
  • 若您稍后使用同一部装置及同一浏览器再次打开同一链接，系统会自动询问："You have a saved draft — continue where you left off?"（是否继续先前的草稿？），点击确认即可恢复所有内容，包括您的签名。
  • 即使您忘记点击保存草稿，系统也会每隔数秒自动在本机保存进度，因此不小心切换页面或关闭页面时，通常仍可在同一装置上恢复。
在其他装置上继续填写。 由于表格无需注册账户，系统只能在您保存草稿的同一装置上自动找到该草稿。若需在其他手机或电脑上继续填写，请点击 📂 Resume Draft，届时会要求输入草稿代码。若您没有此代码，请直接联络 MyEden Group 人力资源部门，他们可根据您的姓名在后台查找草稿并告知您代码。

E. 本地备份（下载／载入 JSON）

除了线上草稿系统外，您也可以在自己的装置上保留一份独立备份：

  • 💾 Download JSON — 将目前表格中已填写的所有内容保存为一个小型文件（副档名为 .json），下载至您装置的常用下载文件夹。此文件仅供您个人保存，不会传送至任何地方。
  • 📂 Load from JSON — 选择先前下载的备份文件，将资料重新载入表格中。此操作会覆盖目前屏幕上的内容，建议在填写开始时使用。

提交申请后，确认页面也会提供 "Download a copy (JSON)" 按钮，方便您保留一份提交内容的个人记录。

F. 提交申请

所有必填栏位填妥后，点击 📤 Submit Application。系统会要求您确认，因为申请一旦提交将无法再修改。确认后将显示成功页面，代表您的申请已成功送出，此时可安全关闭页面。

第二部分 — 人力资源／管理员指南

A. 登入系统

打开人力资源后台链接（通常与申请表格链接不同，一般以 /hr.html 结尾）。使用管理员提供的电邮及密码登入。登入状态会在使用期间自动维持并更新，使用完毕后（尤其是在共用电脑上）请点击 Sign Out 登出。

员工账户由管理员直接建立，系统不开放自行注册。

B. 后台总览

登入后会看到三个分页（"员工权限管理"仅管理员可见）：

  • 📥 Applications（申请列表） — 所有已提交的申请，按最新提交时间排序。
  • 📝 Drafts（草稿列表） — 求职者尚未提交、仍在填写中的表格。
  • 👥 Manage Staff（员工权限管理）（仅限管理员）— 设定每位员工可执行的操作权限。

可随时使用搜索框，依申请人姓名筛选目前分页内容。

C. 申请列表（Applications）

每份申请以卡片形式显示，包含申请人姓名、申请职位、提交日期，以及状态标签（PENDING 待处理 或 PROCESSED 已处理）。点击卡片即可打开完整申请内容，除申请人所填写的资料外，还包含以下的内部使用栏位。

D. 内部使用栏位与聘书详情

请滚动至绿色的 "For Office Use Only"（仅供内部使用） 区块（仅具备相关权限的员工可见）。生成聘书前，请完成以下资料：

  • Approved Job Title（核准职称） — 从营运／行政类别列表中选择，或选择 Other 手动输入职称。
  • Job Title Level（职级） — 选定职称后，此列表会自动显示该职务的职涯发展等级（例如：幼稚园助理 → 幼稚园导师 → 高级幼稚园导师 → 幼稚园主任 → 幼稚园总监）。下方的 Resolved Position（最终职位） 会确认聘书上实际显示的职称。
  • Work Location（工作地点） — 从 MyEden Group 各分行地址中选择，或勾选 Other location 手动输入地址。
  • Letter Date（聘书日期） — 默认为当日日期，可自行更改。
  • Reporting Officer's Name（直属上司姓名）及职称、Salary in Wording（薪金大写）（根据薪金数字自动生成，可编辑），以及其他聘任相关栏位（入职日期、薪金、工作时间等）。
  • Appendix 1（附录一：职务说明） — 系统会根据所选的职称与职级自动填入标准职务说明。勾选 "Manually fill / edit Appendix 1" 可解锁并自行编写或修改内容；取消勾选则会恢复为标准内容。
页面下方会即时显示完整聘书预览，随着您填写以上栏位同步更新，方便在打印前先行核对内容。

E. 保存、状态与删除

  • 💾 Save Changes（页面顶部）或 💾 Save Draft（底部操作栏）— 两者功能相同，都会将内部资料的修改保存至申请记录，可依当前滚动位置选择较方便的按钮使用。
  • Mark Processed（标记为已处理） — 切换申请状态标签；再次点击（此时按钮显示为 Reopen）即可恢复为待处理状态。
  • 🗑 Delete（删除） — 永久删除该申请记录，仅具备删除权限的员工可使用，且系统会先要求确认。

F. 草稿列表（Drafts）

此分页显示所有求职者尚未提交的草稿，包括姓名、联络方式，以及一组草稿代码。此代码可让求职者在其他装置上继续填写：若有求职者致电或留言表示需在新手机上继续填写，可在此依姓名查找并告知对方代码。此处打开的草稿为只读模式，仅供查看进度及协助求职者，无法编辑内容。

G. 打印与导出 PDF

  • 🖶 Print / Save PDF — 打印完整的申请表格内容，包括内部使用栏位及完整的职务说明文字。此打印选项刻意不包含聘书预览部分。
  • 📋 Print Appointment Letter — 仅打印（或透过浏览器打印对话框另存为 PDF）正式的聘书内容，包含信头、完整合约条款、主要条款表格，以及附录一职务说明。
若需要 PDF 文件而非实体打印，请在打印对话框中选择"另存为 PDF"选项。

H. 本地备份（下载／载入 JSON）

除线上记录外，您也可以针对目前打开的申请、草稿或已载入的文件，保存或还原本地备份：

  • 💾 Download JSON（位于页面顶部操作栏）— 将目前打开的记录保存为 .json 文件至您的电脑。
  • 📂 Load from JSON（位于后台顶端）— 打开文件选择窗口，载入先前下载的备份文件。载入后的记录会清楚标示为 "LOADED FROM FILE (not linked to database)"（从文件载入，未连结数据库） — 您可以查看、打印及生成聘书，但由于并非连结至实际数据库记录，保存／状态／删除功能将无法使用。

I. 员工权限管理（仅限管理员）

管理员可见 Manage Staff（员工权限管理） 分页，列出所有员工账户，并可逐项勾选权限：

权限项目控制内容
View applications（查看申请）是否可查看申请列表。
Edit Office Use（编辑内部资料）是否可填写或修改内部使用栏位。
Generate letters（生成聘书）是否可使用聘书区块及打印聘书功能。
Change status（更改状态）是否可将申请标记为已处理／待处理。
Delete records（删除记录）是否可删除申请或草稿记录。

权限变更会立即生效，无需该员工重新登入。管理员自动拥有所有权限，且系统会在管理员尝试取消自己的管理员权限时，先弹出确认提示，避免误操作。

J. 建立员工账户

新的员工账户须直接在 Supabase 项目控制台中建立（并非在人力资源后台内进行），因为此操作需要管理员级别的数据库权限：

  1. 登入 Supabase 项目控制台。
  2. 前往 Authentication → Users → Add user → Create new user。
  3. 输入员工的电邮地址及密码，并保持勾选 Auto Confirm User。
  4. 该账户会自动出现在员工权限管理分页中，并附有安全的默认权限设置，供您进一步调整。

K. 常见问题（人力资源／管理员）

问题处理方法
登入时显示 "Email not confirmed"（电邮未确认）该账户建立时未勾选 Auto Confirm。请在 Supabase 的 SQL Editor 中执行：update auth.users set email_confirmed_at = now() where email_confirmed_at is null;
Supabase 中的 "Send confirmation email" 功能失败此为预期情况 — 系统已刻意关闭自行注册功能，该设置同时也会导致此电邮功能无法使用，请改用上方的 SQL 方法处理。
忘记员工账户密码可直接在 Supabase 中重设：Authentication → Users → 选择该用户 → Reset Password。
重新打开申请后，附录一内容被还原请确认在输入自定内容前已勾选 "Manually fill / edit Appendix 1"，并于填写后点击 Save Changes 保存 — 该勾选状态现已与内容一并保存。
打印出的聘书版面错位或显示不完整请务必以浏览器本身的打印／另存为 PDF 功能查看排版效果；使用其他 PDF 阅读器或转换工具可能会呈现不同的排版结果。
